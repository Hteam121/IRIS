## 22:30 — [Fix] Stable code signing so TCC permissions persist across rebuilds
- Symptom: Screen Recording / Mic permission re-prompts on EVERY launch; toggling the
  System Settings switch + relaunch doesn't stick; the app behaves as if it has no
  permission even though an "IRIS" entry exists (toggled on) in the list.
- Root cause: the project signed ad-hoc (`CODE_SIGN_IDENTITY: "-"`). An ad-hoc signature's
  codesign *designated requirement* is a bare per-build `cdhash`, so every rebuild looks like
  a brand-new app to macOS TCC → the saved grant no longer matches → re-prompt, while the old
  (now-orphaned) hash lingers in the Settings list still toggled on.
- Fix: switched `project.yml` to MANUAL signing with a Development cert already in the
  keychain, selected by SHA-1 (`D9280BD59FC5E00B14EA754F9759C308C361E5C2` = "Apple
  Development: shaikhatim3@gmail.com (566559DMB8)"). Real team/OU is `2ZY9CXJ2SP` (the
  `566559DMB8` in the cert CN is the Apple-ID UID, not the team). Added
  `PROVISIONING_PROFILE_SPECIFIER: ""`, `CODE_SIGNING_REQUIRES_DEVELOPMENT_PROFILE: NO`,
  `CODE_SIGN_INJECT_BASE_ENTITLEMENTS: NO` so a local Mac app signs without a profile/account.
- Verified the new designated requirement is STABLE (cert + bundle-id based, not a cdhash):
  `identifier "com.iris.app" and anchor apple generic and certificate leaf[subject.CN] =
  "Apple Development: shaikhatim3@gmail.com (566559DMB8)" …`. `xcodebuild` macOS Debug
  **BUILD SUCCEEDED**; signature shows Authority = Apple Development, TeamIdentifier 2ZY9CXJ2SP.
- Cleared the orphaned grants for a single clean re-grant: `tccutil reset ScreenCapture /
