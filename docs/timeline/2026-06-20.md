## Menu-bar Settings access + interruption

## 14:00 — [UI] Add Settings… and Interrupt menu items to the eye-icon menu
- `StatusBarItem` only had "Toggle IRIS" and "Quit"; `AppDelegate.onSettings` was wired but
  nothing in the menu ever invoked it, so the API-key window was unreachable. Added a
  "Settings…" item (⌘,) and an "Interrupt IRIS" item (⌘.) plus their callbacks.
- Files: `IRIS/UI/StatusBarItem.swift`.

## 14:10 — [Integration] Global hotkey + menu interrupt
- Added ⌥⎋ (Option+Escape) global hotkey (global + local `NSEvent` monitors) and a
  `handleInterruptRequest()` that cancels the in-flight command, stops TTS, and resets the orb.
  This is the reliable barge-in since the mic is muted during TTS (voice barge-in only works
  while *thinking*, not while *speaking*).
- Files: `IRIS/AppDelegate.swift`.

## 14:20 — [UI] Fix SettingsView build errors
- The drafted `SettingsView.swift` declared `private typealias Settings = IRISSettings`, which
  collided with the global `Settings` config struct (invalid redeclaration) and cascaded into
  the `NSWindow.styleMask` errors. Removed the alias and referenced the config as `IRISSettings`
  throughout. Build now succeeds.
- Files: `IRIS/UI/SettingsView.swift`.
