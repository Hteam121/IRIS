## Realtime "Jarvis/Cluely" core (OpenAI Realtime API) — major rebuild

Replaced the one-shot command router with a continuous speech-to-speech conversation on the OpenAI
Realtime API. The model holds context, reasons, calls tools to actually DO things (reusing all the
existing actions), sees the screen, and can control the Mac. Plan:
~/.claude/plans/for-iris-to-work-fizzy-lagoon.md.

### New
- `IRIS/Realtime/RealtimeAudio.swift` — one AVAudioEngine: mic → 24 kHz mono PCM16 upstream;
  AVAudioPlayerNode plays the model's 24 kHz audio (converted to the engine's native output format
  to avoid the -10875 format errors). Tries voice-processing/AEC when `echoCancellation` is on
  (full-duplex barge-in), else a plain engine; on this Mac AEC is flaky (-10875 / starved input),
  so **default is plain engine + half-duplex**.
- `IRIS/Realtime/RealtimeSession.swift` — WebSocket to `wss://api.openai.com/v1/realtime?model=
  gpt-realtime` (Bearer; **no** `OpenAI-Beta` header — that selects the disabled beta shape). Sends
  the **GA `session.update`** (nested `audio.input/output`, `format {type:audio/pcm, rate:24000}`,
  server VAD, tools). Streams mic, plays `response.output_audio.delta`, shows transcript, dispatches
  `response.function_call_arguments.done` → tool → `function_call_output` + `response.create`.
  Half-duplex mic gating while IRIS speaks (when AEC off). Auto-reconnect; audio-start retry.
- `IRIS/Realtime/RealtimeTools.swift` — tool schemas + dispatch to AppLauncher/FolderOpener/
  WebSearch/TerminalLauncher/AgentManager/LocalAnswers/ComputerControl, plus `look_at_screen`
  (ScreenCapture + gpt-4o vision).
- `IRIS/Actions/ComputerControl.swift` — CGEvent type/click/key + Accessibility prompt.
- `IRIS/Core/IRISLog.swift` — file logger at `~/.iris/iris.log` (NSLog doesn't surface via
  `log show` for this app). Tail it to debug the realtime session.

### Modified

- `Settings` (additive) — `realtimeEnabled`, `realtimeModel` (gpt-realtime), `realtimeVoice`

- `Core.swift` (additive) — `@Published var transcript` (live caption). `OverlayView` shows it.

### Wake-gating + idle pause + hide + Spotlight folders (follow-up)
- **Wake-gated, not always-streaming:** IRIS starts ASLEEP (local `WakeWordDetector` listening for
  "hey iris", `onWakeDetected` → `AppDelegate.wakeUp`); only then does `RealtimeSession` connect +
  stream. Verified: at launch iris.log is empty and traffic is flat (~400 B) — no OpenAI streaming
  until the wake word. Big cost win.
- **15s idle pause:** `RealtimeSession` idle timer (reset on speech/response events) fires after
  `idlePauseSeconds` (default 15) → `onIdleTimeout` → `AppDelegate.goToSleep` (stops the stream,
  restarts the wake listener).
- **Overlay hides after IRIS talks:** on `response.done`, status → `.idle` + `scheduleHide` clears
  the caption ~4s later (cancelled if new speech/response arrives).
- **Find folders anywhere:** `IntentRouter` now falls back to Spotlight (`mdfind`,
  `spotlightFolder`) to locate a named folder nested anywhere on the Mac (prefers under home,
  shallower paths; skips Library/caches/node_modules), instead of stopping at the Desktop level.

### Side-task voice (no overlap) + better path resolution (follow-up)
- **No more two voices at once:** background-agent results no longer play through the separate TTS
  while the realtime voice is talking. `AgentManager.onAnnouncement` routes results to the MAIN
  realtime voice (`RealtimeSession.announce` → injects a system item + `response.create`, queued
  until the current turn ends via `responseActive`/`flushAnnouncements`) when a conversation is
  active; falls back to the TTS `Speaker` only when IRIS is asleep (no main voice to clash with).
- **Folders/paths resolve anywhere:** `IntentRouter.resolveFolderArgument` (used by the realtime
  `open_folder`/`open_terminal` tools) checks a real path first (exists?), then phrase parsing, then
  a bare-name disk + **Spotlight** search — so a bare folder name like "rerun" is found wherever it
  lives (previously Spotlight only ran for "in X" phrasing). If still not found, IRIS says so
  instead of silently opening the wrong place. Verified `mdfind` finds nested folders on this Mac.
